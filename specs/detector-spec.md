# Spec: detection signals & confidence scoring

**File:** `detector.py`
**Status:** Implemented (Milestones 3–4) — Implementation Notes filled in below

---

## Purpose

Estimate whether a piece of submitted text was written by a human or generated by an AI tool,
using two independent signals, and combine them into a single calibrated confidence score
(`p_ai ∈ [0, 1]`) plus an attribution band. This module is the detection pipeline; everything
downstream (label, audit log, appeal) depends on the score it produces.

---

## Input / Output Contract

### `llm_signal(text)` — Signal 1 (semantic)

**Input:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `text` | `str` | The submitted content |

**Output:** `dict`

| Key | Type | Description |
|-----|------|-------------|
| `"p_ai"` | `float` | Probability the text is AI-generated, `[0, 1]` (1 = certainly AI) |
| `"rationale"` | `str` | One-sentence justification from the model |

### `stylometric_signal(text)` — Signal 2 (structural)

**Output:** `dict`

| Key | Type | Description |
|-----|------|-------------|
| `"style_score"` | `float` | Structural AI-likeness, `[0, 1]` (higher = more uniform = more AI-like) |
| `"reliable"` | `bool` | `False` when the text is too short to measure meaningfully |
| `"metrics"` | `dict` | Raw metrics for debugging (`sentence_cv`, `type_token_ratio`, `punct_density`, …) |

### `combine_signals(llm_score, style_score, style_reliable=True)` — scoring

**Output:** `dict`

| Key | Type | Description |
|-----|------|-------------|
| `"p_ai"` | `float` | Combined calibrated confidence, `[0, 1]` |
| `"attribution"` | `str` | One of: `"likely_ai"`, `"uncertain"`, `"likely_human"` |

### `score_to_attribution(p_ai)` — band mapping

`float → str` (one of the three attributions).

---

## Design Decisions

---

### Signal definitions

**Signal 1 — LLM classifier (semantic):**
```
Captures whether the text READS as AI-generated when judged holistically: tone, voice, fluency,
even hedging, and the over-structured / over-polite quality large models tend to produce.
Output is a graded probability, not a binary flag.

Blind spot: framing and lightly edited AI. A human writing in a polished, formal register can
read as "AI," and AI text a human has lightly edited can read as "human." The signal is also
non-deterministic and can drift between calls.
```

**Signal 2 — Stylometric heuristics (structural):**
```
Captures how UNIFORM the writing is, computed in pure Python from three metrics:
  - sentence-length burstiness, as coefficient of variation (stdev / mean of words per sentence)
  - type-token ratio (unique words / total words) — vocabulary diversity
  - punctuation density (marks / word)
AI text is statistically smoother; human writing is bursty and irregular.

Blind spot: length-sensitive and genre-blind. On short inputs the statistics are meaningless;
it mis-scores deliberately uniform human writing (formal abstracts, repetition-heavy poetry,
technical specs) as AI-like.
```

---

### Scoring approach

```
Each signal emits a score in [0, 1]. combine_signals() produces the calibrated p_ai with a
weighted blend plus a disagreement adjustment:

  p_ai = LLM_WEIGHT * llm_score + STYLE_WEIGHT * style_score          (0.6 / 0.4 by default)

The LLM is weighted higher because the holistic read is the stronger single indicator;
stylometrics is the structural cross-check.

Disagreement adjustment — when the semantic and structural signals strongly conflict, the honest
answer is "uncertain," not a confident average:

  d = |llm_score - style_score|
  if d > 0.4:
      pull  = min((d - 0.4) / 0.6, 1.0)        # 0..1 as disagreement grows past 0.4
      p_ai += (0.5 - p_ai) * (0.5 * pull)      # pull toward 0.5, up to 50% at max disagreement

This is the primary defense against false positives (e.g. a repetition-heavy human poem where
the LLM says "human" but stylometrics says "AI").
```

---

### Stylometric metric mapping

```
Each raw metric is converted to an "AI-likeness" sub-score in [0, 1], then weighted:

  ai_var   = 1 - min(cv / 0.7, 1)                  # CV 0 -> 1.0 (uniform/AI), >=0.7 -> 0 (bursty/human)
  ai_ttr   = clamp((0.85 - ttr) / (0.85 - 0.55))   # ttr >= 0.85 -> human, <= 0.55 -> AI
  ai_punct = 1 - min(|density - 0.12| / 0.12, 1)   # ~0.12 marks/word reads machine-steady

  style_score = 0.65 * ai_var + 0.20 * ai_ttr + 0.15 * ai_punct

Burstiness (ai_var) carries the most weight because it is the most reliable discriminator;
punctuation the least. Coefficient of variation is used instead of raw standard deviation so the
metric is comparable across short and long texts.
```

---

### Confidence thresholds (the bands)

```
score_to_attribution(p_ai):
    p_ai >= AI_THRESHOLD (0.80)    -> "likely_ai"
    p_ai <= HUMAN_THRESHOLD (0.30) -> "likely_human"
    otherwise                      -> "uncertain"

The 0.80 bar is deliberately high. Asserting "AI" requires strong evidence so borderline cases
degrade to "uncertain" rather than falsely accusing a human writer. This is not a binary flip at
0.5: a 0.51 and a 0.95 produce different labels.
```

---

### Short-text handling

```
stylometric_signal() flags reliable = False when the text has fewer than 3 sentences OR fewer
than 25 words (MIN_RELIABLE_SENTENCES / MIN_RELIABLE_WORDS). Below that, burstiness and vocabulary
diversity cannot be measured meaningfully.

When style_reliable is False, combine_signals() re-weights to 0.85 LLM / 0.15 stylometric — the
system leans on the LLM and distrusts the noisy structural score, widening effective uncertainty.
```

---

### LLM prompt structure

**System message:**
```
You are an AI-content detector for a creative-writing platform. Assess whether the supplied text
was written by a human or generated by an AI tool. Judge holistically: tone, voice, fluency,
hedging, and over-uniform structure. Respond with ONLY a JSON object of the form
{"p_ai": <number between 0 and 1>, "rationale": "<one short sentence>"}. p_ai is the probability
the text is AI-generated: 1.0 = certainly AI, 0.0 = certainly human.
```

**User message:**
```
{text}
```

The call uses `temperature=0` for repeatability and `response_format={"type": "json_object"}` so
the model returns parseable JSON directly.

---

### Fallback behavior

```
llm_signal(): the JSON response is parsed; p_ai is read with a default of 0.5 if the key is
missing, then clamped to [0, 1] (the model occasionally returns slightly out-of-range values).
The rationale defaults to an empty string.

Defaulting a missing/garbled score to 0.5 (uncertain) is the safe choice — it neither asserts AI
(false positive) nor clears the text (false negative); it routes the decision into the uncertain
band where the label is non-committal and the appeal path is available.
```

## Implementation Notes

**One result that surprised you — input, the score you expected, the score it produced, and why:**

```
The "clearly AI" short formal essay scored llm=0.80 but combined to only 0.66 -> "uncertain",
not "likely_ai" as I first expected. Reason: on a ~45-word sample the stylometric signal scored it
only 0.44 (rich vocabulary, moderate sentence variation), and the 0.80 bar requires both signals
to agree. This is the asymmetry working as designed — the system reserves "likely_ai" for cases
where the structural evidence concurs (e.g. the long, templated essay scored 0.82). It surprised me
at first, then became a documented design property rather than a bug.
```

**One change you made to the scoring after seeing the first calibration run, and what it fixed:**

```
The first stylometric implementation mapped RAW sentence-length standard deviation to a score.
Running tests/test_scoring.py showed every short sample clustered around 0.18–0.34 and the signal
barely separated AI from human. I switched the burstiness metric to coefficient of variation
(stdev / mean, length-normalized) and re-weighted the three metrics to 0.65 / 0.20 / 0.15. After
the change, clearly-AI vs clearly-human separated cleanly (0.44 vs 0.17) and the combined scores
spanned all three bands.
```
